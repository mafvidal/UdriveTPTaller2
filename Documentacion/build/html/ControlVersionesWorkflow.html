Control de versiones y workflow¶

Para el control de versiones se utilizo GitHub

Para ello se crearon tickets, asignando las distintas tareas a desarrollar. Estos tickets si dividieron en las siguientes etiquetas:

  • Cliente el cual hace referencia a las tareas a desarrollar en el cliente.
  • Servidor el cual hace referencia a las tareas a desarrollar en el servidor.
  • Configuracion el cual hace referencia a las herramientas a configurar en el proyecto.
  • Bugs el cual hace referencia a los bugs a corregir en el proyecto.
